# V4 theoretical framework: symmetry-adapted registry theory for bilayer 1T'-ReS2

## 1. Motivation

The V4 model treats bilayer 1T'-ReS2 as a registry-controlled ferroic and optoelectronic system.  The central microscopic variable is not an abstract scalar polarization, but the interlayer sliding-registry vector

```text
u = (u_a, u_b)
```

where `u_a` follows the low-barrier/easy sliding channel and `u_b` describes the transverse/hard sliding coordinate.  The out-of-plane polarization, SHG tensor, Raman/ULF Raman fingerprints, anisotropic excitonic PL, and electrical/PV responses are then treated as different projections of the same hidden registry coordinate.

The model is designed to be deeper than a phenomenological curve-fitting workflow, but still honest about its status: it is a symmetry-constrained, calibration-ready phenomenological theory unless its registry energy surface, Berry-phase polarization, optical tensors, and transport coefficients are replaced by DFT or same-device experimental fits.

## 2. Physical foundation

### 2.1 Registry as the structural order parameter

For sliding ferroelectric bilayers, the ferroelectric state is generated by relative layer translation or stacking transformation. Therefore, the polarization should be written as a function of interlayer registry:

```text
P_z = P_z(u_a, u_b)
```

rather than as a free scalar order parameter.  V4 enforces this by constructing symmetry-adapted basis functions and requiring ferroelectric contributions to be odd under the operation that maps one polar state to the opposite polar state.

### 2.2 Free-energy hierarchy

The recommended V4 free energy is

```text
F(u, E_z, T, n, c_v)
  = U_reg(u, n, c_v)
  + U_local(u, T)
  - E_z P_z(u, n, c_v)
  + U_defect(u, n, c_v)
```

where:

- `U_reg` is the periodic stacking-registry energy.
- `U_local` is a local Landau-like expansion around the easy sliding channel.
- `-E_z P_z` gives field coupling.
- `U_defect` captures doping/vacancy-induced barrier and polarization renormalization.

The current repository provides a demo-level Fourier registry surface.  Quantitative switching barriers require DFT stacking-energy maps and NEB pathways.

### 2.3 Symmetry-adapted polarization decomposition

V4 decomposes polarization into three channels:

```text
P_z(u) = P_Landau(u) + P_Berry-like(u) + P_charge-transfer(u)
```

1. `P_Landau`: local odd sliding-channel component.
2. `P_Berry-like`: periodic registry component that mimics Berry-phase polarization modulation.
3. `P_charge-transfer`: conservative interlayer charge-transfer proxy.

The charge-transfer term is not a substitute for Berry-phase DFT.  It is a calibration target that can later be fitted using charge-density-difference maps, layer-resolved Bader charge, electrostatic potential offsets, or direct ferroelectric measurements.

### 2.4 Optical response hierarchy

The optical response should be separated into three physically distinct contributions:

```text
Observable(u, E_laser)
  = structural registry term
  + polarization-odd term
  + exciton-resonance term
```

This separation is important because not every stacking-sensitive optical change is ferroelectrically switchable.  Some observables may be even under polarization reversal, some odd, and some mixed.

### 2.5 Raman and ULF Raman

High-frequency Raman tensors probe intralayer vibrational anisotropy and exciton-resonant enhancement.  Ultralow-frequency Raman modes are more directly tied to interlayer shear and breathing dynamics, and therefore should carry the strongest structural weight in registry inversion.

Recommended joint loss:

```text
Loss_total(u)
  = w_SHG Loss_SHG(u)
  + w_ULF Loss_ULF(u)
  + w_HF Loss_HF_Raman(u)
  + w_PL Loss_PL(u)
  + w_IV Loss_IV(u)
```

V4 default weighting should prioritize ULF Raman and SHG for registry identification, while PL primarily constrains excitonic readout.

### 2.6 Excitonic PL

ReS2 has strongly anisotropic excitonic optical response.  Therefore, V4 keeps the two PL peaks as separate branches:

```text
X1: E1, Gamma1, oscillator1, theta1, DOLP1
X2: E2, Gamma2, oscillator2, theta2, DOLP2
```

A peak-resolved PL interpretation is only reliable when

```text
|E_X2 - E_X1| > max(Gamma_X1, Gamma_X2)
```

If this criterion is not satisfied, the model should report an integrated-emission response rather than claiming independent X1/X2 excitonic axes.

### 2.7 Transport and photovoltaic response

V4 explicitly decomposes photocurrent into parity channels:

```text
J_total
  = J_dark_even
  + J_dark_odd
  + J_shift_out_odd
  + J_shift_in_even
  + J_shift_in_mixed
```

Only odd candidate terms can be interpreted as switchable by ferroelectric reversal, and even/mixed terms must not be described as direct ferroelectric memory readouts unless experimentally verified.

## 3. V4 code modules

The V4 upgrade adds the following modules:

```text
functions/symmetry_adapted_registry_basis.m
functions/sliding_polarization_v4.m
functions/transport_pv_response_v4.m
functions/validate_model_v4.m
```

### 3.1 symmetry_adapted_registry_basis.m

Builds polynomial and Fourier registry basis functions, then classifies each basis term as:

```text
odd / even / mixed
```

under the operation connecting opposite polar states.  The default operation is `u -> -u`, but users can replace it by defining:

```matlab
p.symmetry.polarTransformMatrix = [m11 m12; m21 m22];
```

### 3.2 sliding_polarization_v4.m

Returns a structured polarization decomposition:

```matlab
out.Pz
out.P_landau
out.P_berry_like
out.P_charge_transfer
out.renormalization
out.claimLevel
out.basisParity
```

This supports deeper physical writing because the model can state whether a simulated polarization arises from the local sliding channel, registry-periodic Berry-like modulation, or charge-transfer proxy.

### 3.3 transport_pv_response_v4.m

Splits transport/PV response into parity channels:

```matlab
tr.J_dark_even
tr.J_dark_odd
tr.J_shift_out_odd
tr.J_shift_in_even
tr.J_shift_in_mixed
tr.photocurrent
tr.parity
```

This is essential for avoiding overclaiming switchable photocurrent.

### 3.4 validate_model_v4.m

Generates:

```text
output/validation_v4/v4_validation_checks.csv
output/validation_v4/v4_polarization_decomposition.csv
output/validation_v4/v4_photocurrent_parity_decomposition.csv
output/validation_v4/MODEL_V4_AUDIT_REPORT.md
```

The validation layer checks:

- whether `Pz` reverses under the polar-partner operation;
- whether odd registry basis functions exist;
- whether ULF Raman changes across registry states;
- whether X1/X2 PL peaks are resolvable;
- whether transport/PV channels are parity-decomposed;
- whether demo-mode quantitative claims are blocked.

## 4. Recommended manuscript-level interpretation

### Defensible high-level statement

```text
We developed a symmetry-adapted registry model in which the interlayer sliding vector serves as a hidden structural coordinate that simultaneously determines ferroelectric polarization, tensorial optical fingerprints, anisotropic excitonic emission, and electrical/PV response channels.
```

### Mechanistic statement

```text
Within this framework, ferroelectric switching is represented as a transition between symmetry-related registry states.  The resulting optical response is not treated as an isolated empirical modulation, but as a registry-dependent tensorial response containing structural, polarization-odd, and exciton-resonant components.
```

### Honest claim boundary

```text
The present implementation provides a symmetry-constrained, calibration-ready phenomenological theory.  Quantitative prediction of switching barriers, coercive fields, absolute polarization, and photovoltaic current requires DFT/NEB or same-device experimental calibration.
```

---

## 5. Recommended next DFT/experiment calibration targets

Priority order:

1. DFT stacking-energy surface `U_reg(u_a, u_b)`.
2. Berry-phase polarization `P_z(u_a, u_b)`.
3. NEB minimum-energy switching path and barrier.
4. ULF Raman shear/breathing frequencies versus registry.
5. SHG tensor components versus registry.
6. X1/X2 exciton energy, oscillator strength, and polarization axes versus registry.
7. Photocurrent tensor components and parity under polar-state reversal.

After steps 1-3, the model can make stronger switching-path and barrier statements.  After steps 4-6, it can support quantitative optical inversion.  After step 7, it can support switchable PV/transport claims.

## 6. Practical use

Run the original model as before:

```matlab
main_run_all_ReS2_sliding_model
```

For V4 validation:

```matlab
p = default_res2_params();
p = apply_research_guided_v3_constraints(p);
report = validate_model_v4(p, fullfile(pwd, 'output', 'validation_v4'));
```

The V4 layer intentionally does not overwrite the original model.  It adds a deeper theoretical scaffold and a stricter audit trail for manuscript-level claims.
